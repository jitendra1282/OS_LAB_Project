## Project Overview

This project extends the **xv6 operating system** (RISC-V version) by adding and modifying system calls in the kernel. xv6 is a small, clean Unix-like teaching OS developed at MIT. The goal is to:

1. Study how system calls are implemented end-to-end in a real OS kernel.
2. Add new system calls for process introspection, inter-process communication (IPC), and shared memory.
3. Demonstrate each system call with a working user-space test program running inside xv6.
4. Document all changes with source code and execution flow.

### What is xv6?
xv6 is a simple, Unix-like operating system written in C for the RISC-V architecture. It implements core OS concepts like:
- Process management (fork, exec, wait, exit)
- Virtual memory with page tables
- File system
- System calls via trap/ecall mechanism
- Spinlocks and sleep locks for synchronization

### System Call Mechanism in xv6
When a user program needs a kernel service, it triggers a **system call** via the RISC-V `ecall` instruction. The flow is:
```
User program
     ↓  calls wrapper function (e.g., getprocinfo())
usys.S (generated by usys.pl)
     ↓  loads syscall number into register a7, executes ecall
Kernel trap handler (trap.c / kernelvec.S)
     ↓  saves registers, enters kernel mode
syscall() in syscall.c
     ↓  looks up function pointer in syscalls[] table by number
sys_xxx() in sysproc.c / shm.c / sysfile.c
     ↓  performs the actual kernel work
Return value placed in a0 register → back to user space
```

```c
// sys_getprocinfo — print info about the current process from kernel.
// No arguments. Returns 0 on success.
uint64
sys_getprocinfo(void)
{

  printf("Process name:  %s\n", p->name);

}
```

## System Call 2 — Shared Memory IPC: `shmget` / `shmattach` / `shmdetach` (Member 1 — Jitendra)

### Purpose
Implements **POSIX-style shared memory** Inter-Process Communication (IPC) inside xv6. Shared memory is the fastest IPC mechanism because data is exchanged by reading and writing to the same physical memory pages — no copying needed.

This implementation adds **three new system calls**:

| Syscall | Number | Signature | Description |
|---------|--------|-----------|-------------|
| `shmget` | 23 | `int shmget(int key, int size)` | Create a new shared segment or get an existing one by key |
| `shmattach` | 24 | `void *shmattach(int shmid)` | Map the segment's physical page into the caller's virtual address space |
| `shmdetach` | 25 | `int shmdetach(int shmid)` | Unmap the segment; free the physical page when the last process detaches |

### Concept: How Shared Memory Works
```
Process A                        Process B
─────────                        ─────────
Virtual Address Space            Virtual Address Space
┌─────────────┐                  ┌─────────────┐
│   code      │                  │   code      │
│   heap      │                  │   heap      │
│ shared page │ ─────────────── │ shared page │
│  (va=0x4000)│        ↑        │  (va=0x5000)│
└─────────────┘        │        └─────────────┘
                        │
               Physical Memory
               ┌─────────────┐
               │  REAL PAGE  │ ← ONE physical page, pa=0x84000
               │(4096 bytes) │
               └─────────────┘

Both virtual addresses map to the same PA via page table entries.
Writing at va=0x4000 in Process A is instantly visible at va=0x5000 in Process B.
```

### Kernel Data Structures

**`kernel/shm.c`** — the shared memory table:
```c
#define NSHMSEGS  16    // kernel supports up to 16 simultaneous shared segments
#define SHM_FREE   0    // slot is available
#define SHM_USED   1    // slot is in use

// One entry in the global shared memory table
struct shmseg {
  struct spinlock lock;   // per-segment spinlock for concurrent access safety
  int     state;          // SHM_FREE or SHM_USED
  int     key;            // user-chosen integer identifier (like a "name")
  uint64  pa;             // physical address of the allocated page (0=none)
  int     refcount;       // count of processes currently attached to this segment
};

// Global table — lives in kernel BSS, initialized by shminit() at boot
static struct shmseg shm_table[NSHMSEGS];
static struct spinlock shm_table_lock;  // protects table-level operations
```

### Implementation — `shminit()`

Called from `kernel/main.c` once during kernel boot, before any process starts:

```c
// kernel/main.c  (line added):
shminit();   // initialize shared memory subsystem

// kernel/shm.c:
void
shminit(void)
{
  initlock(&shm_table_lock, "shm_table");
  for (int i = 0; i < NSHMSEGS; i++) {
    initlock(&shm_table[i].lock, "shmseg");
    shm_table[i].state    = SHM_FREE;
    shm_table[i].key      = 0;
    shm_table[i].pa       = 0;
    shm_table[i].refcount = 0;
  }
}
```

### Implementation — `sys_shmget()`

Creates a new shared segment or returns the ID of an existing one matching the given key.

```c
// kernel/shm.c
// Arguments:  key  (arg 0) — user-chosen integer key
//             size (arg 1) — requested size; must be > 0 and <= PGSIZE (4096)
// Returns:    shmid (index 0..15) on success, -1 on error
uint64
sys_shmget(void)
{
  int key, size;
  argint(0, &key);    // read first syscall argument
  argint(1, &size);   // read second syscall argument

  if (key < 0 || size <= 0 || size > PGSIZE)
    return -1;

  acquire(&shm_table_lock);  // lock the whole table for the search

  // --- Phase 1: Check if a segment with this key already exists ---
  for (int i = 0; i < NSHMSEGS; i++) {
    if (shm_table[i].state == SHM_USED && shm_table[i].key == key) {
      release(&shm_table_lock);
      return i;   // found — return existing shmid, no allocation needed
    }
  }

  // --- Phase 2: Allocate a new slot ---
  for (int i = 0; i < NSHMSEGS; i++) {
    if (shm_table[i].state == SHM_FREE) {
      acquire(&shm_table[i].lock);    // lock the individual slot
      shm_table[i].state    = SHM_USED;
      shm_table[i].key      = key;
      shm_table[i].refcount = 0;

      // Allocate exactly one physical page (4096 bytes) from the kernel allocator
      char *pa = kalloc();
      if (pa == 0) {                  // out of memory
        shm_table[i].state = SHM_FREE;
        release(&shm_table[i].lock);
        release(&shm_table_lock);
        return -1;
      }
      memset(pa, 0, PGSIZE);          // zero-initialize for security
      shm_table[i].pa = (uint64)pa;

      release(&shm_table[i].lock);
      release(&shm_table_lock);
      return i;   // return new shmid
    }
  }

  release(&shm_table_lock);
  return -1;   // table is full (16 concurrent segments already in use)
}
```

### Implementation — `sys_shmattach()`

Maps the shared segment's physical page into the calling process's virtual address space.

```c
// kernel/shm.c
// Argument:  shmid (arg 0) — segment ID returned by shmget()
// Returns:   virtual address (uint64) where the page was mapped, or (uint64)-1 on error
uint64
sys_shmattach(void)
{
  int shmid;
  argint(0, &shmid);

  if (shmid < 0 || shmid >= NSHMSEGS)
    return -1;

  struct shmseg *seg = &shm_table[shmid];
  acquire(&seg->lock);

  // Validate: segment must exist and have a physical page
  if (seg->state != SHM_USED || seg->pa == 0) {
    release(&seg->lock);
    return -1;
  }

  struct proc *p = myproc();

  // Choose virtual address: page-aligned start just above current process heap
  uint64 va = PGROUNDUP(p->sz);

  // Insert a new PTE in the process page table mapping va → seg->pa
  // Flags: PTE_R (readable) | PTE_W (writable) | PTE_U (user-accessible)
  if (mappages(p->pagetable, va, PGSIZE, seg->pa, PTE_R | PTE_W | PTE_U) < 0) {
    release(&seg->lock);
    return -1;
  }

  // Grow the process's recorded size to include this new page
  p->sz = va + PGSIZE;

  seg->refcount++;    // one more process is using this segment
  release(&seg->lock);

  return va;   // user receives this as a regular void* pointer
}
```

### Implementation — `sys_shmdetach()`

Removes the mapping from the process's page table and frees the physical page if no other process is attached.

```c
// kernel/shm.c
// Argument:  shmid (arg 0) — segment ID to detach from
// Returns:   0 on success, -1 on error
uint64
sys_shmdetach(void)
{
  int shmid;
  argint(0, &shmid);

  if (shmid < 0 || shmid >= NSHMSEGS)
    return -1;

  struct shmseg *seg = &shm_table[shmid];
  acquire(&seg->lock);

  if (seg->state != SHM_USED || seg->pa == 0) {
    release(&seg->lock);
    return -1;
  }

  struct proc *p = myproc();

  // Walk every page in the process's virtual address space.
  // Find the PTE whose physical address matches seg->pa.
  int found = 0;
  for (uint64 va = 0; va < p->sz; va += PGSIZE) {
    pte_t *pte = walk(p->pagetable, va, 0);
    if (pte == 0 || !(*pte & PTE_V))
      continue;
    uint64 phys = PTE2PA(*pte);     // extract PA from PTE
    if (phys == seg->pa) {
      // Unmap: clear the PTE, do NOT free the physical page (it's shared!)
      uvmunmap(p->pagetable, va, 1, 0);
      found = 1;
      break;
    }
  }

  if (!found) {
    release(&seg->lock);
    return -1;   // this process wasn't attached
  }

  seg->refcount--;

  // Last detach: release the physical page back to the kernel allocator
  if (seg->refcount <= 0) {
    kfree((void *)seg->pa);
    seg->pa       = 0;
    seg->state    = SHM_FREE;
    seg->key      = 0;
    seg->refcount = 0;
  }

  release(&seg->lock);
  return 0;
}
```

### User Test Program

**`user/testshm.c`** — full source:
```c
#include "kernel/types.h"
#include "kernel/stat.h"
#include "user/user.h"

#define SHM_KEY  42     // shared key — both parent and child use this
#define SHM_SIZE 4096   // one full page (max supported size)

int
main(void)
{
  printf("=== Shared Memory Demo (shmget/shmattach/shmdetach) ===\n");

  // Step 1: Parent creates (or finds) shared segment with key=42
  int shmid = shmget(SHM_KEY, SHM_SIZE);
  if (shmid < 0) { printf("shmget failed\n"); exit(1); }
  printf("[Parent] shmget(key=%d) -> shmid=%d\n", SHM_KEY, shmid);

  // Step 2: Parent maps the segment into its address space
  char *shared = (char *)shmattach(shmid);
  if ((uint64)shared == (uint64)-1) { printf("shmattach failed\n"); exit(1); }
  printf("[Parent] shmattach -> va=0x%p\n", shared);

  // Step 3: Parent writes a message into shared memory
  char *msg = "Hello from Parent via Shared Memory!";
  int i;
  for (i = 0; msg[i] != '\0'; i++) shared[i] = msg[i];
  shared[i] = '\0';
  printf("[Parent] wrote: \"%s\"\n", shared);

  // Step 4: Fork — child inherits parent's page table (copy-on-write)
  int pid = fork();
  if (pid == 0) {
    // Child: look up the existing segment by the same key
    int cshmid = shmget(SHM_KEY, SHM_SIZE);
    char *cshared = (char *)shmattach(cshmid);  // gets its own VA mapping
    printf("[Child]  shmattach -> va=0x%p\n", cshared);
    printf("[Child]  read:  \"%s\"\n", cshared);  // reads parent's message

    // Child writes a reply
    char *reply = "Hello back from Child!";
    int j;
    for (j = 0; reply[j] != '\0'; j++) cshared[j] = reply[j];
    cshared[j] = '\0';

    shmdetach(cshmid);   // unmap (refcount goes from 2→1, page stays alive)
    printf("[Child]  shmdetach OK — exiting\n");
    exit(0);
  }

  // Parent waits for child, then reads the reply child left
  int xstate;
  wait(&xstate);
  printf("[Parent] child exited with status %d\n", xstate);
  printf("[Parent] reads after child wrote: \"%s\"\n", shared);

  shmdetach(shmid);  // unmap (refcount goes from 1→0, physical page freed)
  printf("[Parent] shmdetach OK\n");
  printf("=== Shared Memory Demo Complete ===\n");
  exit(0);
}
```

### Design Decisions

| Decision | Reason |
|----------|--------|
| Fixed-size table of 16 segments | Simple; avoids dynamic allocation in kernel; sufficient for a teaching OS |
| One physical page (4096 bytes) per segment | Matches xv6's page size; simpler than multi-page segments |
| Two-level locking (table lock + per-segment lock) | Table lock protects the search/alloc; per-segment lock protects attach/detach |
| `uvmunmap(..., 0)` — do NOT free on detach | The page is shared; only free when `refcount` drops to 0 |
| VA placement at `PGROUNDUP(p->sz)` | Avoids overlapping with existing process memory; extends heap upward |
| Walk page table in detach to find VA | Avoids storing per-process VA in the segment struct (keeps data structure simple) |

### Error Handling

| Condition | Return |
|-----------|--------|
| `key < 0` or `size <= 0` or `size > PGSIZE` | `shmget` returns -1 |
| Segment table is full (16 used) | `shmget` returns -1 |
| `kalloc()` fails (out of memory) | `shmget` returns -1, slot rolled back to FREE |
| Invalid `shmid` (< 0 or ≥ 16) | `shmattach`/`shmdetach` return -1 |
| Segment not in USED state | `shmattach`/`shmdetach` return -1 |
| `mappages()` fails | `shmattach` returns -1 |
| Process was not attached to segment | `shmdetach` returns -1 |

## System Call 3 — `fork_with_priority` (Member 2 — MD.salman)

### Purpose
`fork_with_priority()` extends the process creation mechanism of the xv6 operating system by introducing priority-based process creation. Unlike the default `fork()` system call, which creates a child process with identical attributes, this custom system call allows assigning a priority value to the newly created process. This lays the foundation for implementing priority-based scheduling.

### Syscall Number
```c
// kernel/syscall.h
#define SYS_fork_with_priority 26
```

### Files Modified / Created

| File | Description |
|------|-------------|
| `kernel/proc.h` | Added `priority` field to the `proc` structure |
| `kernel/proc.c` | Initialized default priority to 60 in `allocproc()` |
| `kernel/sysproc.c` | Wrote `sys_fork_with_priority()` — the kernel-side handler |
| `kernel/syscall.h` | Assigned syscall number `26` for `SYS_fork_with_priority` |
| `kernel/syscall.c` | Registered syscall in the dispatch table |
| `user/user.h` | Declared user-level function `int fork_with_priority(int);` |
| `user/usys.pl` | Added `entry("fork_with_priority");` syscall stub generation |
| `user/test_priority.c` | **NEW** — Test program for priority-based fork |
| `Makefile` | Added `$U/_test_priority` to the `UPROGS` build list |

### Kernel Implementation Details

**1️⃣ Process Structure Update** (`kernel/proc.h`)
```c
struct proc {
  // ... existing fields ...
  int priority;
};
```

**2️⃣ Default Initialization** (`kernel/proc.c` in `allocproc`)
```c
p->priority = 60;
```

**3️⃣ System Call Implementation** (`kernel/sysproc.c`)
```c
uint64 sys_fork_with_priority(void) {
  int priority;
  argint(0, &priority);

  int pid = sys_fork();

  if(pid < 0)
    return -1;

  if(pid == 0){
    struct proc *p = myproc();
    p->priority = priority;
  }

  return pid;
}
```

### User Test Program

**`user/test_priority.c`**
```c

int main() {
  int pid = fork_with_priority(10);

  if(pid == 0){
    printf("Child process (priority = 10)\n");
  } else {
    printf("Parent process\n");
  }

  exit(0);
}
```

```
Return pid  →  back to user space
```

## System Call 4 — `getpinfo` (Member 3)

### 📌 Overview

This system call provides a detailed snapshot of all active processes in the xv6 operating system.

Unlike basic process inspection, `getpinfo()` exposes kernel-level scheduling data to user space, enabling monitoring and analysis of process behavior. This is essential for implementing and evaluating advanced scheduling algorithms.

### 🎯 Objective

- Provide visibility into the process table.
- Enable analysis of CPU scheduling behavior.
- Support debugging and performance evaluation.
- Bridge kernel internals with user programs.

### ⚙️ System Call Definition

```c
int getpinfo(struct pstat *ps);
```

### 🧠 Functionality

- Copies process information from kernel to user space using `copyout()`.
- Fills a structure containing:
  - Process IDs
  - Process states
  - Priority values
  - CPU usage ticks
  - Whether a slot is in use
- Returns: `0` (success) or `-1` (failure).

### 📂 Files Modified

| File | Description |
|------|-------------|
| `kernel/proc.h` | Added `int ticks;` for tracking CPU usage |
| `kernel/proc.c` | Initialized and updated CPU tick counters |
| `kernel/sysproc.c` | Implemented syscall logic securely using `copyout()` |
| `kernel/syscall.h` | Added syscall number `SYS_getpinfo` |
| `kernel/syscall.c` | Registered syscall in dispatch table |
| `user/user.h` | Declared function and struct mapping |
| `user/usys.pl` | Added perl entry script for Trap generation |
| `kernel/pstat.h` | Defined the `pstat` datastructure |
| `user/test_pinfo.c` | Verification Program |
| `Makefile` | Compiled new test program |

### 🧩 Data Structure

```c
#define NPROC 64

struct pstat {
  int pid[NPROC];
  int state[NPROC];
  int priority[NPROC];
  int ticks[NPROC];
  int inuse[NPROC];
};
```

### 🧪 Test Program `user/test_pinfo.c`

```c

int main() {

  getpinfo(&ps);

  for(int i = 0; i < NPROC; i++){
    if(ps.inuse[i]){
      printf("PID: %d | State: %d | Priority: %d | Ticks: %d\n",
        ps.pid[i],
        ps.state[i],
        ps.priority[i],
        ps.ticks[i]);
    }
  }

  exit(0);
}
```

## System Call 5 - Modified 'kill' with Signal support(Member 3 - Nithya Harshini)

### Purpose:
The 'Kill()' system call is modified to support signal-based process termination instead of directly killing the process.This demonstrates:
- Extending an existing system call in xv6
- Introducing a basic signal implementation
- Understanding  process lifecycle and trap handling
- Interaction between proc structure and kernel execution

### Original Behaviour (before Modification)

The original kill() system call works by setting flag in the process structure.



**Code**

**`(kernel/proc.c)``** :
```c
p->killed = 1;

if(p->state == SLEEPING){
  p->state = RUNNABLE;
}
```
 

### Original Execution Flow 
```
     kill(pid)
        │
        │  system call
        ↓
sys_kill() → kkill()
        │
        │  sets flag
        ↓
  p->killed = 1
        │
        ↓
    usertrap()
        │
        │  checks flag
        ↓
  if(killed(p))
        │
        ↓
     exit()
```



### Limitation
- Only supports direct termination
- No support for multiple signal types
- Cannot extend behaviou like pause/resume

### Files modified

| File | What Changed |
|-------| ------------------------|
| `kernel/proc.h` | Declared signal_pending and signal_type |
| `kernel/proc.c` | Initialized signal_pending and signal_type and also modified kkill() |
| `kernel/trap.c` | Added signal handling logic |
| `user/test_kill.c` | New test program added to user folder |
| `Makefile` | Added $U/_test_kill |

### Modifications
1.Process Structure Modification

**`kernel/proc.h`**
```c
int signal_pending;   // whether signal exists
int signal_type;      // type of signal
```
2.Initiialization in allocproc()

**`kernel/proc.c`**
```c
p->signal_pending = 0;
p->signal_type = 0;
```
3.Modification kill implementation

**`kernel/proc.c`** ---->  `kkill()`
```c
p->signal_pending = 1;   // signal exists
p->signal_type = 9;      // SIGKILL
p->killed = 1;
```
4.Implementation in trap.c 

**`kernel/trap.c`**
```c
if(p && p->signal_pending){
  if(p->signal_type == 9){
    printf("Process %d killed by signal\n", p->pid);
    p->signal_pending = 0;
    kexit(-1);
  }
}
```

### Execution flow (After Modification)
```
   kill(pid)
        │
        │  system call
        ↓
sys_kill() → kkill()
        │
        │  set signal
        ↓
  signal_pending = 1
  signal_type = 9
        │
        ↓
    usertrap()
        │
        │  check signal
        ↓
if(signal_pending)
        │
        ↓
process terminated using kexit()
```


### User Test program
**`user/test_kill.c`**

## How to Add More System Calls (For Teammates)

Every system call in xv6 follows the same 7-step checklist. Assign your syscall number from the list below.

### Allocated Syscall Numbers
| Member | Name | Num |
|--------|------|-----|
| 1 (Jitendra) | `getprocinfo` | 22 |
| 1 (Jitendra) | `shmget` | 23 |
| 1 (Jitendra) | `shmattach` | 24 |
| 1 (Jitendra) | `shmdetach` | 25 |
| 2 (MD.salman) | `fork_with_priority` | **26** |
| 3 (Neelamber) | `getpinfo` | **27** |
| 4 (Nithya) | `kill` | **6** |
| **Member 5** | your syscall | **29** |
| **Member 6** | your syscall | **30** |

### Step-by-Step Guide

**Step 1** — Pick a name (e.g., `mysyscall`) and add its number to `kernel/syscall.h`:
```c
#define SYS_mysyscall 26
```

**Step 2** — Add the kernel handler. Either add a function to `kernel/sysproc.c` or create a new `kernel/mysyscall.c` file:
```c
uint64
sys_mysyscall(void)
{
  // read arguments with argint(), argstr(), argaddr()
  int n;
  argint(0, &n);
  // ... do kernel work ...
  return 0;   // 0 = success, -1 = error
}
```

**Step 3** — If you created a new `.c` file, add it to `kernel/defs.h`:
```c
// mysyscall.c
uint64  sys_mysyscall(void);
```

**Step 4** — Register it in `kernel/syscall.c`:
```c
// Declaration (with the other externs)
extern uint64 sys_mysyscall(void);

// Dispatch table (inside syscalls[] array)
[SYS_mysyscall] sys_mysyscall,
```

**Step 5** — Declare it for user programs in `user/user.h`:
```c
int mysyscall(int n);   // adjust signature to match your syscall
```

**Step 6** — Add the ecall stub in `user/usys.pl`:
```perl
entry("mysyscall");
```

**Step 7** — Write a test program `user/test_yourname.c` and add it to `Makefile`:
```makefile
# In UPROGS:
$U/_test_yourname \
```
